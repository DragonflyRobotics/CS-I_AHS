2.1 What do you notice about the relationship between the first-half (top four rows) of the full-adder as compared to all of the rows of the half-adder? --> The first half is identical to the truth table of the half-adder.

2.2 Why is this true? --> This is because when we don't have a carried value to worry about so the first half will have the exact same output as a half-adder that cannot handle carried values.

3.1 Why does the least significant bit position use a half-adder rather than a full-adder? --> Because it doesn't have to worry about anything being carried from the previous place. Since it has no carried values to add to the sum, it can simple do the half addition. If we used the full-adder instead, it would be missing the carry argument.

3.2 Assume that proper inputs are applied for all bits in numbers A and B. Will the correct output from S be available instantaneously? If not, why not? --> It won't be available instantaneously because the sum must wait for the previous adders to finish in order to do the computation. This is because each full-adder needs the carry value from the previous added.
